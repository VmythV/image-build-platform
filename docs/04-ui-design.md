# 前端管理界面设计

## 1. 设计目标

前端是面向镜像维护者和平台管理员的后台管理系统。设计重点是高效、清晰、可排错，而不是营销展示。

目标：

- 让管理员快速完成主机、仓库和系统配置。
- 让镜像维护者清楚地看到镜像项目、起始镜像、版本节点和分支关系。
- 让构建任务的状态、日志和失败原因足够直观。
- 让高风险操作有明确提示。
- 让界面具备国际化开源产品的视觉风格。

## 2. UI 技术基线

前端技术：

- React。
- TypeScript。
- Vite。
- TanStack Query。
- shadcn/ui。
- Radix UI。
- Tailwind CSS。
- lucide-react。
- @xyflow/react。
- Monaco Editor。

设计语言：

- 中性配色。
- 高信息密度。
- 清晰表格。
- 稳定侧边导航。
- 操作优先级明确。
- 图标辅助操作。
- 少装饰，少大色块。

## 3. 全局布局

后台使用固定侧边栏 + 顶部栏 + 内容区布局：

```text
┌──────────────┬──────────────────────────────────────────────┐
│ Sidebar      │ Topbar                                       │
│              ├──────────────────────────────────────────────┤
│ Navigation   │ Page content                                 │
│              │                                              │
│              │                                              │
└──────────────┴──────────────────────────────────────────────┘
```

### 3.1 侧边导航

一级导航：

- Dashboard。
- Build Hosts。
- Registries。
- Image Projects。
- Build Tasks。
- Artifacts。
- Settings。
- Help。

导航规则：

- 当前页面高亮。
- 构建中任务可以在 `Build Tasks` 上显示轻量计数。
- 主机离线或仓库异常不在侧边栏制造强提醒，放在 Dashboard 和对应页面中展示。

### 3.2 顶部栏

顶部栏包含：

- 当前页面标题。
- 主要操作按钮。
- 全局搜索入口，MVP 可先只做入口。
- 当前用户菜单。
- 主题切换，MVP 可选。
- 语言切换，MVP 可先预留。

### 3.3 内容区

内容区按页面类型分为：

- 列表页。
- 详情页。
- 工作台页。
- 设置页。
- 帮助页。

列表页默认使用表格。工作台页可以使用三栏布局，例如版本图页面。

## 4. 通用交互规范

### 4.1 表格

表格必须支持：

- loading 状态。
- empty 状态。
- error 状态。
- 分页。
- 关键字段排序，MVP 可按页面选择实现。
- 行内主要状态。
- 行操作菜单。

表格不直接堆大量按钮。常用动作放在行尾，低频动作放入更多菜单。

### 4.2 表单

表单规则：

- 必填字段明确标记。
- 保存前做前端基础校验。
- 后端校验错误要映射回字段。
- 密码、token、SSH 私钥不回显。
- 危险配置旁边显示提示。

表单按钮：

- 主按钮：保存、创建、提交构建。
- 次按钮：取消、测试连接。
- 危险按钮：删除、禁用、取消构建。

### 4.3 详情面板

复杂详情优先使用右侧抽屉：

- 版本节点详情。
- 构建任务摘要。
- 镜像产物详情。
- 主机检测结果。
- 仓库检测结果。

抽屉适合在不离开当前上下文的情况下查看更多信息。

### 4.4 状态展示

状态展示需要统一：

| 状态 | 视觉 |
| --- | --- |
| success | 绿色徽标 |
| failed | 红色徽标 |
| running | 蓝色徽标或 spinner |
| queued | 灰色或中性色徽标 |
| warning | 黄色徽标 |
| disabled | 灰色弱化 |

状态文字必须可读，不能只依赖颜色。

### 4.5 危险操作

危险操作需要二次确认：

- 删除主机。
- 删除仓库。
- 删除版本节点，MVP 可不提供。
- 取消构建。
- 删除本地镜像产物。
- 关闭关键主机。

确认弹窗需要说明影响范围。

## 5. 页面设计

### 5.1 登录页

用途：

- 用户登录后台。

字段：

- 用户名。
- 密码。

状态：

- 登录失败。
- 登录过期。
- 首次初始化完成后跳转登录。

验收：

- 登录成功进入 Dashboard。
- 未登录访问后台页面跳转登录页。

### 5.2 首次初始化页

首次启动且无管理员账号时展示。

字段：

- 管理员用户名。
- 管理员密码。
- 确认密码。

说明：

- 提示该账号拥有系统最高权限。
- 提示初始化后可在用户管理中修改。

验收：

- 创建成功后进入登录页或直接进入 Dashboard。

### 5.3 Dashboard

用途：

- 提供平台整体状态概览。

区块：

- 构建任务状态统计。
- 最近失败任务。
- 构建主机状态。
- 镜像仓库状态。
- 近期构建趋势，MVP 可用简单列表或计数。
- 快捷入口：添加主机、添加仓库、创建镜像项目、发起构建。

展示字段：

- 运行中任务数。
- 排队任务数。
- 今日成功数。
- 今日失败数。
- 在线主机数。
- 异常仓库数。

交互：

- 点击统计卡进入对应筛选后的列表。
- 点击失败任务进入任务详情。

#### 5.4.1 主机列表

字段：

- 主机名称。
- 连接方式。
- 地址。
- 架构。
- Docker 版本。
- 状态。
- 当前并发。
- 最大并发。
- 标签。
- 最近检测时间。

操作：

- Add Host。
- Check。
- Edit。
- Disable/Enable。
- Delete。
- View Details。

筛选：

- 状态。
- 架构。
- 连接方式。
- 标签。

#### 5.4.2 新增主机

连接方式：

- Local Docker。
- SSH。

Local Docker 字段：

- 名称。
- Docker socket 路径，默认 `/var/run/docker.sock`。
- 最大并发。
- 标签。

SSH 字段：

- 名称。
- 地址。
- 端口。
- 用户名。
- 认证方式：SSH key，MVP 优先。
- SSH 私钥。
- Docker 命令路径，默认 `docker`。
- 最大并发。
- 标签。

检测结果：

- SSH 连通性。
- Docker 可用性。
- 架构。
- Docker 版本。
- 磁盘空间。
- 最近错误。

高风险提示：

- Local Docker 使用宿主机 Docker socket 时，需要提示安全风险。
- SSH 凭据需要提示最小权限原则。

字段：

### 5.7 Version Graph

版本图是核心工作台页面。

布局：

```text
┌─────────────────────────────────────────────────────────────┐
│ Project / Branch / Status filters / Actions                │
├───────────────┬───────────────────────────────┬─────────────┤
│ Project list  │ Git-style version graph       │ Node detail │
│ Filters       │                               │ drawer      │
└───────────────┴───────────────────────────────┴─────────────┘
```

顶部区域：

- 当前镜像项目。
- 当前起始镜像。
- 分支筛选。
- 状态筛选。
- 创建版本。
- 创建分支。
- 发起构建。

图节点显示：

- 版本号或 tag。
- 分支。
- 构建状态。
- 创建时间。

节点视觉：

- 成功：绿色状态点。
- 失败：红色状态点。
- 构建中：蓝色状态点。
- 未构建：灰色状态点。
- 归档：弱化显示。

节点交互：

- 点击：打开右侧详情。
- 双击：进入版本详情页，MVP 可选。
- 悬停：显示简短摘要。
- 右键或更多菜单：创建子版本、创建分支、发起构建、比较差异。

图交互：

- 缩放。
- 拖拽画布。
- 定位到最新节点。
- 按分支折叠。
- 适配视图。

右侧详情面板：

- 版本号。
- 分支。
- 父节点。
- 来源节点。
- Dockerfile。
- 表单配置快照。
- 说明。
- 最近构建记录。
- 关联镜像产物。
- 操作按钮。

详情操作：

- Edit Dockerfile。
- Create Child Version。
- Create Branch。
- Build。
- Compare。
- View Builds。

## 6. 空状态和错误状态

### 6.1 空状态

每个核心列表必须有可操作空状态：

- 无主机：显示 Add Host。
- 无仓库：显示 Add Registry。
- 无镜像项目：显示 Create Project。
- 无构建任务：提示从版本节点发起构建。
- 无产物：提示构建成功后会出现产物。

### 6.2 错误状态

错误信息需要包含：

- 发生在哪里。
- 错误摘要。
- 下一步建议。
- 查看详细日志入口，若适用。

示例：

- `Docker daemon is not reachable. Check whether /var/run/docker.sock is mounted.`
- `Registry login failed. Verify username, token, and registry endpoint.`
- `Host architecture arm64 does not match requested amd64.`

## 7. 响应式设计

MVP 优先支持桌面端。

最低目标：

- 1366px 宽度下完整可用。
- 1024px 宽度下可用。
- 小屏下侧边栏可折叠。

版本图、Dockerfile 编辑器和日志页不强制优化到手机端。手机端可以只保证基础查看。

## 8. 国际化

MVP 文案可以先使用英文 UI，中文文档同步维护。

前端需要预留 i18n 结构：

- 文案集中管理。
- 不把页面文案散落在复杂组件内部。
- 状态枚举有统一翻译映射。

后续支持：

- English。
- 简体中文。

## 9. 可访问性

基础要求：

- 表单字段有 label。
- 按钮有可理解文本或 tooltip。
- 图标按钮必须有 aria-label。
- 弹窗和抽屉可通过键盘关闭。
- 状态不能只依赖颜色表达。

shadcn/ui 和 Radix UI 的组件能力应保留，不要破坏焦点管理。

## 10. 前端验收清单

MVP 前端完成时应满足：

- 用户可以完成首次初始化和登录。
- 用户可以添加并检测本机构建主机。
- 用户可以添加并检测 SSH 构建主机。
- 用户可以添加并检测镜像仓库。
- 用户可以创建镜像项目并设置起始镜像。
- 用户可以进入 Git-style 版本图。
- 用户可以创建版本节点和分支。
- 用户可以编辑 Dockerfile。
- 用户可以发起构建。
- 用户可以在任务详情看到实时日志。
- 用户可以在构建完成后看到镜像产物。
- 所有核心列表都有 loading、empty、error 状态。
- 所有危险操作有确认。
- 登录过期后可以回到登录页。
